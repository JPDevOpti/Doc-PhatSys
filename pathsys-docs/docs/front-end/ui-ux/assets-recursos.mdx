## ¿Qué son los assets y recursos?

Los **assets** y **recursos** son todos los elementos estáticos que utiliza PathSys para su funcionamiento visual y funcional. Incluyen iconos, imágenes, estilos CSS, fuentes y otros archivos que no son código JavaScript/TypeScript pero son esenciales para la experiencia de usuario.

### ¿Por qué son importantes?

- **Identidad visual**: Logos, iconos e imágenes que representan la marca LIME
- **Consistencia**: Recursos reutilizables que mantienen la coherencia visual
- **Performance**: Organización optimizada para carga eficiente
- **Mantenibilidad**: Estructura clara que facilita la gestión de recursos

## Estructura de assets en PathSys

### Organización de archivos

```
Front-End/src/assets/
├── FavIco.ico                  ← Favicon del sitio
├── icons/                      ← Sistema de iconos SVG
│   ├── index.ts               ← Exportaciones centralizadas
│   ├── SVG/                   ← Archivos SVG originales
│   ├── HomeIcon.vue           ← Componentes de iconos
│   ├── DashboardIcon.vue      ← (70+ iconos Vue)
│   └── ... (más iconos)
├── images/                     ← Imágenes estáticas
│   ├── Logo-LIME-Fondo.jpeg   ← Logo LIME con fondo
│   ├── Logo-LIME-NoFondo.png  ← Logo LIME sin fondo
│   ├── LOGO-LIME-Inicial.png  ← Logo inicial LIME
│   ├── Banner_HAMA.png        ← Banner Hospital Alma Máter
│   └── Banner_UDEA.png        ← Banner Universidad de Antioquia
└── styles/                     ← Sistema de estilos CSS
    ├── main.css               ← Archivo principal de estilos
    ├── base/                  ← Estilos base del sistema
    ├── components/            ← Estilos de componentes
    └── vendor/                ← Estilos de librerías externas
```

## Sistema de estilos CSS

### Estructura modular de estilos

```
Front-End/src/assets/styles/
├── main.css                    ← Punto de entrada principal
├── base/                       ← Estilos fundamentales
│   ├── variables.css          ← Variables CSS del sistema
│   ├── fonts.css              ← Configuración de fuentes
│   ├── reset.css              ← Reset de estilos del navegador
│   └── globals.css            ← Estilos globales de la aplicación
├── components/                 ← Estilos específicos de componentes
│   ├── utilities.css          ← Clases de utilidad personalizadas
│   ├── navigation.css         ← Estilos de navegación
│   ├── buttons.css            ← Estilos de botones
│   ├── forms.css              ← Estilos de formularios
│   ├── tables.css             ← Estilos de tablas
│   └── auth.css               ← Estilos de autenticación
└── vendor/                     ← Estilos de librerías externas
    ├── apexcharts.css         ← Estilos para gráficos
    ├── flatpickr.css          ← Estilos para selector de fechas
    ├── fullcalendar.css       ← Estilos para calendario
    ├── swiper.css             ← Estilos para carruseles
    └── datepicker.css         ← Estilos adicionales de fechas
```

## Mejores prácticas para assets

### Organización de archivos

**Iconos:**
- Mantener consistencia en el viewBox (24x24px)
- Usar `currentColor` para flexibilidad de color
- Incluir `aria-hidden="true"` para iconos decorativos
- Exportar desde un índice centralizado

**Imágenes:**
- Usar formatos apropiados (PNG para transparencia, JPEG para fotos)
- Optimizar para web (compresión adecuada)
- Proporcionar versiones responsive cuando sea necesario
- Nombrar archivos de manera descriptiva

**Estilos:**
- Mantener separación de responsabilidades
- Usar variables CSS para valores reutilizables
- Importar en orden lógico (reset → base → components → vendor)
- Documentar variables y clases personalizadas
