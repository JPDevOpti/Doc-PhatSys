## ¿Qué es un Layout?

Un **layout** es la estructura visual base que define cómo se organizan los elementos en todas las páginas de la aplicación. En PathSys, el layout funciona como un marco que contiene elementos fijos (como el header y sidebar) y un área dinámica donde cambia el contenido según la página que esté viendo el usuario.

### ¿Por qué es importante?

- **Consistencia visual**: Todas las páginas mantienen la misma estructura
- **Navegación intuitiva**: El usuario siempre sabe dónde encontrar las funciones principales
- **Eficiencia**: Los elementos comunes (menú, header) no necesitan recargarse al cambiar de página
- **Experiencia de usuario**: Reduce la carga cognitiva al mantener elementos familiares

El **header** es la barra superior que contiene:

- **Logo de LIME**: Identidad visual de la organización
- **Información del usuario**: Nombre, rol y foto de perfil
- **Menú de usuario**: Acceso rápido al perfil y configuración

**¿Por qué está siempre visible?**
- Proporciona contexto de identidad al usuario
- Permite acceso rápido a funciones críticas (notificaciones, perfil)
- Mantiene la marca institucional presente en todo momento

### Área de Contenido Principal

Esta es la zona donde se renderiza el contenido específico de cada página. Aquí aparecen:

- **Páginas de módulos**: Lista de casos, formularios, reportes, etc.
- **Cards informativos**: Tarjetas con métricas y resúmenes
- **Formularios**: Interfaces para crear y editar información
- **Tablas y listados**: Datos organizados en filas y columnas

### ¿Qué son las Cards?

Las **cards** (tarjetas) son componentes visuales que presentan información específica de manera organizada:

- **Métricas clave**: Número de casos pendientes, completados, etc.
- **Gráficos**: Visualizaciones de tendencias y estadísticas
- **Acciones rápidas**: Botones para funciones frecuentes
- **Estado del sistema**: Indicadores de salud del laboratorio

**¿Por qué usar cards?**
- **Escaneabilidad**: La información se puede leer rápidamente
- **Organización**: Cada métrica tiene su propio espacio visual
- **Interactividad**: Permiten acciones directas sobre la información
- **Responsividad**: Se adaptan bien a diferentes tamaños de pantalla

## Navegación por Roles

### ¿Cómo funciona?

La navegación se adapta automáticamente según el rol del usuario:

- **Patólogos**: Ven módulos de aprobación, reportes avanzados y estadísticas
- **Residentes**: Acceso a casos asignados y herramientas de aprendizaje
- **Auxiliares**: Funciones operativas como registro de pacientes y pruebas
- **Administradores**: Herramientas de configuración y gestión del sistema

### ¿Por qué es importante?

- **Seguridad**: Cada usuario solo ve las funciones que puede usar
- **Eficiencia**: Reduce la confusión al mostrar solo opciones relevantes
- **Experiencia personalizada**: Cada rol tiene una interfaz optimizada para sus necesidades

## Responsividad

### Adaptación a dispositivos

El layout se adapta automáticamente según el dispositivo:

- **Desktop**: Sidebar siempre visible, layout completo
- **Tablet**: Sidebar colapsible, contenido optimizado
- **Móvil**: Sidebar como overlay, navegación simplificada
