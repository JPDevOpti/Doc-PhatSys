## ¿Qué son los listados?

Los **listados** son componentes especializados en PathSys que permiten buscar y seleccionar elementos de catálogos específicos del sistema. A diferencia de las tablas que muestran datos para consulta, los listados son componentes de entrada que facilitan la selección de valores predefinidos como entidades, patólogos, pruebas, municipios o regiones del cuerpo.

### ¿Por qué son importantes?

- **Selección guiada**: Proporcionan opciones predefinidas y válidas
- **Búsqueda eficiente**: Permiten encontrar elementos específicos rápidamente
- **Consistencia de datos**: Aseguran que se seleccionen valores válidos del sistema
- **Experiencia de usuario**: Interfaz intuitiva para selección de elementos

## Ubicación de los listados en el código

Los componentes de listados en PathSys están organizados en una estructura modular:

### Estructura de archivos

```
Front-End/src/shared/components/ui/lists/
├── EntityList.vue           ← Listado de entidades de salud
├── PathologistList.vue     ← Listado de patólogos
├── TestList.vue            ← Listado de pruebas médicas
├── MunicipalityList.vue    ← Listado de municipios
├── BodyRegionList.vue      ← Listado de regiones del cuerpo
├── DiseaseList.vue         ← Listado de enfermedades
├── MethodSelector.vue      ← Selector de métodos
└── index.ts                ← Exportaciones centralizadas
```

## Funcionalidades comunes de los listados

### Búsqueda en tiempo real

Todos los listados incluyen funcionalidad de búsqueda:

- **Filtrado instantáneo**: Los resultados se filtran mientras se escribe
- **Búsqueda por múltiples campos**: Nombre, código, descripción
- **Resultados relevantes**: Prioriza coincidencias exactas
- **Sin resultados**: Mensaje claro cuando no se encuentran elementos

### Estados de carga

Los listados manejan diferentes estados:

- **Cargando**: Spinner mientras se obtienen los datos
- **Con datos**: Lista poblada con opciones
- **Sin datos**: Mensaje cuando no hay elementos disponibles
- **Error**: Indicación cuando ocurre un problema

### Validación y selección

- **Campo requerido**: Indicador visual con asterisco (*)
- **Validación en tiempo real**: Feedback inmediato sobre selección
- **Estado de selección**: Visualización clara del elemento seleccionado
- **Limpieza de selección**: Opción para deseleccionar

## Mejores prácticas en listados

### Organización y estructura

- **Jerarquía lógica**: Organizar elementos de general a específico
- **Agrupación temática**: Agrupar elementos relacionados
- **Orden alfabético**: Facilitar la búsqueda manual
- **Información relevante**: Mostrar datos útiles para la selección

### Experiencia de usuario

- **Búsqueda intuitiva**: Placeholders descriptivos y útiles
- **Resultados claros**: Información suficiente para tomar decisiones
- **Feedback visual**: Estados de carga, error y éxito
- **Accesibilidad**: Navegación por teclado y lectores de pantalla

### Rendimiento

- **Carga bajo demanda**: Solo cargar datos cuando sea necesario
- **Filtrado eficiente**: Optimizar búsquedas en grandes listas
- **Cache de resultados**: Evitar recargas innecesarias
- **Paginación**: Para listas muy extensas

### Validación y consistencia

- **Campos requeridos**: Indicadores claros de obligatoriedad
- **Validación en tiempo real**: Feedback inmediato
- **Mensajes de error**: Descriptivos y accionables
- **Consistencia visual**: Mismo comportamiento en todos los listados

Esta estructura de listados asegura que los usuarios de PathSys puedan seleccionar información de manera eficiente, clara y consistente en todo el sistema.
